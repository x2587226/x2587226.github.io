使用Hexo快速搭建属于自己的博客

  • hexo 博客

前言

很早之前就一直有个想法，搭建一个属于自己的博客，来记录自己的工作和生活，分享人生旅途中的各种经历和感悟。自己是一个有轻度拖延症的人，所以这个想法迟迟没有实现。新的一年到了，给自己定下的第一个目标，就是把个人博客搭建起来，于是就有了这么一段经历。
首先，我是一个不懂编程的产品汪，对于开发技术只是略知皮毛（甚至皮毛都谈不上），所以对于不懂编程的童鞋不必太有压力，使用Hexo搭建这个博客其实并不需要太多的编程知识，只要你耐心看完以下内容，一定可以轻松完成。
这里简单说一下Hexo对于Wordpress的优势：

  • Hexo可以直接托管在github上，不用自己租赁虚拟服务器，简单粗暴的说法就是省钱！
  • Hexo生成的是静态页面，有利于SEO；
  • Hexo主题比较简洁，是本人比较喜欢的风格；
    其他的不多说，立即开始正题！

1.安装Git

git是一款免费、开源的分布式版本控制系统，我下载的版本是2.7.3，下载后一路next，按照默认流程按照完成即可，具体用法我们后面会讲到。

2.安装node.js

hexo是用node.js写的，所以我们需要安装node.js，我下载的版本是4.4.0，同样一路next即可。

3.安装Hexo

在任何空白的地方（比如桌面空白处），点击右键，我们发现多了两个按钮，一个是Git GUI Here，一个是Git Bush Here，点击Git Bush Here，打开了Git命令行窗口。

输入以下指令，点击回车开始安装Hexo：

npm install -g hexo

稍等一会之后，Hexo就安装完成了，输入hexo -v 可以查看当前版本号。

PS：如果报错，可以尝试更换taobao的npm源，输入以下命令：

npm install -g cnpm –registry=https://registry.npm.taobao.org

再重新输入 npm install -g hexo 进行安装。

4.安装npm

npm是Node.js的包管理和分发工具，用它可以很快的找到特定服务要使用的包，进行下载、安装以及管理已经安装的包。
同样的，我们在Git命令行窗口中输入以下命令，点击回车开始安装npm：

npm install

安装完成后，我们可以输入npm -v查看当前版本号。

5.创建hexo文件夹

安装完基本工具包后，我们需要一个本地文件夹来存放网页文件，在任何你喜欢的地方新建一个文件夹，命名为hexo（如E:\hexo），新建完成后进入该文件夹中，右键打开Git Bush，输入以下命令：

hexo init

执行指令后，当前文件夹（E:\hexo）中会生成以下内容：
├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
| ├── _drafts
| └── _posts
└── themes
这样搭建博客所需要的文件我们就都安装好了，接下来我们需要安装一些必要的插件。

6.安装hexo插件

因为最新版本的hexo在安装过程中有一些必要插件没有绑定安装，需要我们进行手动安装，下面是个人比较推荐安装的一些插件的集合。

在hexo文件夹中打开Git Bush，直接复制以下代码执行：

npm install hexo-generator-index –save
npm install hexo-generator-archive –save
npm install hexo-generator-category –save
npm install hexo-generator-tag –save
npm install hexo-server –save
npm install hexo-deployer-git –save
npm install hexo-deployer-heroku –save
npm install hexo-deployer-rsync –save
npm install hexo-deployer-openshift –save
npm install hexo-renderer-marked@0.2 –save
npm install hexo-renderer-stylus@0.2 –save
npm install hexo-generator-feed@1 –save
npm install hexo-generator-sitemap@1 –save

安装完成后，我们就可以开始使用hexo的几个简单指令搭建博客啦。

7.预览本地博客

执行以下命令，成功后可登录localhost:4000查看效果

hexo s

我们看到了hexo默认主题下的博客页面，虽然不那么美观，不过好歹能看了！是不是有点成就感了？
那么接下来我们准备将生成的静态页面部署到Github上，这样才能让大家在网络上浏览你的博客。

8.与github建立关联

第一步，打开github注册页面，注册一个github账户，这里我们举个例子，假设我们注册一个username为xuxuancheng的账号，过程很简单，按照提示注册完成即可。
（已有账号的朋友可以跳过此步骤，直接登录即可）

第二步，我们要在github上创建一个仓库来存放网页文件，成功登录后，点击右上角黑色+号，选择new repository，开始创建。
Repository name记得必须与你的username相对应，比如我们的username为xuxuancheng，那么Repository name就设置为xuxuancheng.github.io，仓库类型选择为public，点击创建即可。

第三步，由于远程代码是基于SSH的，所以我们需要对SSH进行相关配置。
先在本地生成SSH公钥，然后添加到Github上面。具体的操作如下：

（1）首先打开Git Bush，设置你的用户名和邮箱，输入指令：

git config –global user.email “xuxuancheng@qq.com”
git config –global user.name “xuxuancheng”

PS：这里的邮箱填写你自己能用的邮箱，用户名可随意填写

（2）输入以下命令生成密钥：

ssh-keygen -t rsa -C “xuxuancheng@qq.com”

密钥生成过程中，会让你输入密码，如下:

Enter passphrase (empty for no passphrase):
Enter same passphrase again:

设置密码，输入的密码不显示（也可以不设置，按三次回车，密码为空）
上述的命令成功后，会得到id_rsa和id_rsa.pub两个文件，可能在C:\Users\Administrator.ssh文件夹里。

（3）用文本编辑器打开ssh.pub文件，拷贝其中的内容，然后登录github，
打开右上角菜单，选择Setting，然后选择SSH keys，选择New SSH key，在Key表单中将刚才复制的全部内容粘贴进去（Title可随意），点击Add SSH key，这样便添加完成了。

（4）如果还不放心，我们可以测试一下，在Git Bush中输入：

ssh -T git@github.com

如果看到以下内容，说明你连接成功了

Hi XXX！You’ve successfully authenticated, but GitHub does not provide shell access.

9.将静态页面部署到github上

我们已经成功和github建立的关联，可以开始准备将本地的页面部署到github上了，在此之前，我们还需要进行一些设置。

找到E:\hexo目录下的_config.yml文件，用文本编辑器打开，在最下面找到deploy，并进行以下修改：（仍以xuxuancheng为例，有xuxuancheng的地方请自行替换成你注册的用户名）

deploy：
type： git
repo： git@github.com:xuxuancheng/xuxuangcheng.github.io.git
branch: master
massage: update

PS：冒号后面记得加一个空格，否则视为格式错误，部署的时候会出错

修改完保存，打开Git Bush，输入以下指令：

hexo g
hexo d

当你看到以下这行字的时候，说明部署成功了！

INFO Deploy done: git

最后，在你的浏览器地址栏中输入xuxuangcheng（你的用户名）.github.io，就可以看到你的博客啦~
至此，我们的搭建博客的基础流程也就到此结束了，下一篇文章我们将提高一下博客的颜值并进行相应的配置。
